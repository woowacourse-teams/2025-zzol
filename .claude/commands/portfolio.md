# 맞춤형 포트폴리오 자동 생성

쫄 프로젝트의 노션 "기여 카탈로그" DB에서 입력된 채용공고(JD)에 가장 적합한 기여·트러블슈팅·자산 항목을 선별하고, 노션에 새 맞춤형 포트폴리오 페이지를 생성합니다.

## 입력

JD: `$ARGUMENTS`

`$ARGUMENTS`가 비어 있으면 사용자에게 JD 입력 방법(붙여넣기/URL/파일 경로)을 묻고 중단한다. JD 외에 회사명·직무명을 명시하지 않았다면 한 번에 함께 받는다.

## 노션 리소스 (하드코딩 — 변경되면 이 파일을 갱신할 것)

- **기여 카탈로그 DB**: `https://www.notion.so/6cde78c7aa5e454c9fff20e892f9aa05`
- **데이터 소스 URL**: `collection://8e41230a-af90-4ce5-810d-f6f4be9887e2`
- **마스터 페이지 (쫄 프로젝트)**: `2f620b265277800abe0bcadee1c4b181`
- **상위 포트폴리오 페이지 (커피빵 프로젝트 포트폴리오)**: `2ae20b26527780bebf2fcebfa9dcd0b8`
- **공통 취업 아카이브 데이터 소스**: `collection://e49c1c4e-0bbd-422f-a56a-429f59eff344`

## 작업 순서

### 1단계 — JD 수집

- URL이면 `WebFetch`로 본문을 가져온다. 회사 채용 페이지가 JS 렌더링으로 본문이 비어 있으면 사용자에게 "본문을 붙여넣어 달라"고 요청한다.
- 로컬 파일 경로(`.txt`, `.md`)면 `Read`로 읽는다.
- 그 외에는 텍스트로 처리한다.

### 2단계 — JD 분석 (요구 역량 추출)

JD를 읽고 다음을 구조화한다:

- **회사명 / 직무명** — 페이지 제목과 톤에 사용
- **회사 도메인/제품** — fit 설명에 사용 (예: 핀테크, 커머스, AI 인프라, 게임 등)
- **필수 기술 스택** — 카탈로그의 `Skills`와 매칭
- **요구 역량 키워드** — 카탈로그의 `JD Tags`와 매칭
- **우대사항** — 가산점 (있으면 별도로 적시)

### 3단계 — 카탈로그 DB 매칭

`mcp__66cb5936-0648-46b2-a771-1036c15a2624__fetch`로 데이터 소스(`collection://8e41230a-af90-4ce5-810d-f6f4be9887e2`)를 가져와 모든 페이지의 properties를 확인하고 매칭한다. 페이지가 많아질 경우 `mcp__66cb5936-0648-46b2-a771-1036c15a2624__search`에 `data_source_url`을 넘겨 의미 기반 검색을 보조 수단으로 쓴다.

각 카탈로그 페이지에 매칭 점수를 부여:

- **Skills 일치** × 3점
- **JD Tags 일치** × 2점
- **Domains 일치** × 1점
- `Priority` 값을 가산점으로 더한다 (이미 1–10)
- 회사 도메인이 명확히 무관한 항목(예: 게임 회사가 아닌데 게임 도메인 자체를 강조하는 톤)은 감점

`Type` 별로 다음 비율을 목표로 상위 항목을 고른다:

- **기여**: 3 – 5건
- **트러블슈팅**: 1 – 2건
- **AI엔지니어링 자산**: JD에 AI/AgenticDev 또는 생산성 키워드가 있을 때 1건
- **한계/개선**: 직무 시니어리티가 시니어/리드급이거나 "성장/회고" 요구가 있을 때 1건

매칭이 약한 항목은 무리하게 포함하지 않는다. 매칭 점수 표는 본문에 그대로 노출하지 말고, 마지막 보고에서만 사용자에게 보여준다.

### 4단계 — 페이지 생성

`mcp__66cb5936-0648-46b2-a771-1036c15a2624__notion-create-pages`로 새 페이지를 생성한다.

- **부모**: `{"type": "page_id", "page_id": "2ae20b26527780bebf2fcebfa9dcd0b8"}` (커피빵 프로젝트 포트폴리오 아래)
- **아이콘**: 🎯
- **제목**: `쫄 프로젝트 — {회사명} {직무명} 맞춤 포트폴리오`
- **본문 구조** (Notion enhanced markdown):

```markdown

```

### 5단계 — 보고

페이지 생성이 끝나면 사용자에게 다음 형식으로 보고:

```

다음에 직접 확인하면 좋을 부분:
  - {예: "이 회사가 DDD를 강조하니, 마스터의 도메인 모델 문서를 보강한 뒤 4.1을 다시 작성하면 더 강해집니다"}
```

## 작성 원칙

- **AI는 제안하고 사람이 결정한다.** 매칭이 애매하면 빼고, 사용자에게 추가 후보로 보고한다. 무리하게 끼워 맞추지 않는다.
- **정량 지표는 절대 과장하지 않는다.** 카탈로그의 Impact 문구·숫자는 그대로 인용한다.
- **회사·직무 톤에 맞춘 재배열은 허용**되지만, 새 사실을 만들어내지 않는다. 카탈로그에 없는 기술/경험은 포트폴리오에 넣지 않는다.
- **이미지 URL**은 마스터 노션의 S3 URL이 만료될 수 있으니, 새 페이지에는 직접 임베드하지 말고 "상세" 링크로 카탈로그 페이지를 가리킨다.
- **개인정보**: JD에 면접관 이름·이메일 같은 민감 정보가 있다면 포트폴리오 본문에 넣지 않는다.
