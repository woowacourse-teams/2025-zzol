# ADR 형식·상태·index 행

`adr` SKILL.md에서 참조하는 형식 상세.

## ADR 형식

```markdown
# NNNN. {제목}

- 날짜: YYYY-MM-DD
- 상태: 승인

## 컨텍스트

왜 이 결정이 필요했는가. 해결해야 했던 문제와 당시 상황을 서술한다.

## 결정

무엇을 선택했는가.

## 고려한 대안

| 대안 | 장점 | 단점 |
|------|------|------|
| ...  | ...  | ...  |

## 트레이드오프

선택한 방안의 장단점과 감수한 것들을 서술한다.

## 결과

이 결정이 코드베이스와 팀에 미치는 영향을 서술한다.
```

## 상태(status) 값

작성·갱신 시점에 맞는 값을 쓴다. 이미 구현된 결정을 밋밋하게 `승인`으로만 두지 않는다.

- `승인` — 결정했으나 아직 미구현 또는 구현 중
- `적용됨` — 코드에 반영 완료
- `보류` — 결정했으나 적용 보류
- `폐기` / `대체됨` — 번복. 사유와 대체 ADR 링크를 본문에 추가한다

개정·후속 PR이 있으면 주석을 붙인다: `승인 (2026-04-30 범위 확장)`, `적용됨 (PR #1289 머지)`, `승인 (Phase 1 완료, #1389)`.

## index.md 행 형식

5개 컬럼을 모두 채운다. 번호는 파일 링크로 건다.

```text
| [NNNN](NNNN-title.md) | 제목 | 상태 | 영향 범위 | 핵심 제약 |
```

- **영향 범위**: 이 결정이 적용되는 패키지 또는 도메인 (예: `global/redis`, `cardgame/`)
- **핵심 제약**: 이 결정으로 지켜야 할 규칙을 한 문장으로 (Claude가 충돌 여부를 판단하는 기준이 된다)

> `index.md`는 `.gitattributes`에서 `merge=union`이라 여러 브랜치가 각자 행을 추가해도 자동 병합된다. 맨 끝에 추가만 하면 된다.
