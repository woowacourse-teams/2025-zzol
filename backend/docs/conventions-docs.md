# 문서 작성 컨벤션

`docs/` 디렉토리 내 Markdown 파일 작성 시 지켜야 할 규칙이다.
이 프로젝트는 markdownlint 검사를 적용하며, 아래 규칙을 위반하면 린트 오류가 발생한다.

## MD040 — 코드 블록에 언어 지정 필수

모든 펜스 코드 블록(` ``` `)에 언어를 명시한다. 언어를 특정할 수 없는 경우 `text`를 사용한다.

````text
# 잘못된 예
```
SELECT * FROM room_session;
```

# 올바른 예
```sql
SELECT * FROM room_session;
```

# 언어 불명확 시
```text
nickname_audit → player_name_audit
```
````

자주 쓰는 언어 식별자: `java`, `sql`, `yaml`, `bash`, `json`, `text`

## MD031 — 코드 블록 앞뒤에 빈 줄 필수

펜스 코드 블록 바로 앞과 바로 뒤에 빈 줄이 있어야 한다.

````text
# 잘못된 예
다음은 예시다:
```sql
SELECT 1;
```
이후 내용

# 올바른 예
다음은 예시다:

```sql
SELECT 1;
```

이후 내용
````

## MD022 — 헤딩 앞뒤에 빈 줄 필수

`#`, `##`, `###` 등 헤딩 바로 앞과 바로 뒤에 빈 줄이 있어야 한다.
문서 첫 줄 헤딩의 앞 빈 줄은 생략 가능하다.

```text
# 잘못된 예
## 컨텍스트
왜 이 결정이 필요했는가.

# 올바른 예
## 컨텍스트

왜 이 결정이 필요했는가.
```
