# 문서 작성 컨벤션

Markdown 파일 작성 시 지켜야 할 규칙과 그 배경을 설명한다.

> **규칙의 단일 진실 원천(SSOT)은 저장소 루트의 `.markdownlint.jsonc`다.**
> 어떤 규칙이 실제로 강제되는지는 항상 그 파일이 정답이며, `Docs CI`(`.github/workflows/docs-ci.yml`)가
> `markdownlint-cli2`로 위반을 차단한다. 이 문서는 규칙을 다시 나열하지 않고, 채택 규칙의 **"왜"와 적용 예시**만 다룬다.
> 검사 대상 파일 범위(globs/ignores)는 `.markdownlint-cli2.jsonc`에 있다.

현재 채택 규칙은 두 묶음이다. 나머지 기본 규칙은 꺼져 있으므로(`"default": false`),
설정 파일에 명시되지 않은 규칙(`MD041` 등)은 **이 프로젝트의 규칙이 아니다**. 규칙을 추가·제거하려면 `.markdownlint.jsonc`를 수정한다.

- **렌더링 규칙** (어기면 렌더링이 깨짐): `MD040`, `MD031`, `MD022`
- **위생 규칙** (자동 수정 가능, 노이즈 방지): `MD047`, `MD012`, `MD007`

로컬에서 검사하려면 저장소 루트에서 다음을 실행한다.

```bash
npx markdownlint-cli2@0.22.1          # 검사 (CI와 동일 버전)
npx markdownlint-cli2@0.22.1 --fix    # 자동 수정 가능한 위반(MD031/MD022/MD047/MD012/MD007) 정리
```

## MD040 — 코드 블록에 언어 지정 필수

**왜:** 언어가 지정되어야 렌더러·뷰어가 문법 강조를 적용하고, 코드 블록의 의도(어떤 언어/형식인지)가 명확해진다.

모든 펜스 코드 블록(` ``` `)에 언어를 명시한다. 언어를 특정할 수 없는 경우(다이어그램, 플로우 트리, 의사 메시지 등) `text`를 사용한다.

````text
# 잘못된 예
```
SELECT * FROM room_session;
```

# 올바른 예
```sql
SELECT * FROM room_session;
```

# 언어 불명확 시
```text
nickname_audit → player_name_audit
```
````

자주 쓰는 언어 식별자: `java`, `sql`, `yaml`, `bash`, `json`, `text`

## MD031 — 코드 블록 앞뒤에 빈 줄 필수

**왜:** 빈 줄이 없으면 일부 렌더러가 코드 블록을 앞뒤 문단과 합쳐 깨뜨린다. 빈 줄로 블록 경계를 분명히 한다.

펜스 코드 블록 바로 앞과 바로 뒤에 빈 줄이 있어야 한다.

````text
# 잘못된 예
다음은 예시다:
```sql
SELECT 1;
```
이후 내용

# 올바른 예
다음은 예시다:

```sql
SELECT 1;
```

이후 내용
````

## MD022 — 헤딩 앞뒤에 빈 줄 필수

**왜:** 빈 줄이 없으면 헤딩이 인접 텍스트와 같은 문단으로 묶여 헤딩으로 렌더링되지 않는 경우가 있다.

`#`, `##`, `###` 등 헤딩 바로 앞과 바로 뒤에 빈 줄이 있어야 한다. 문서 첫 줄 헤딩의 앞 빈 줄은 생략 가능하다.

```text
# 잘못된 예
## 컨텍스트
왜 이 결정이 필요했는가.

# 올바른 예
## 컨텍스트

왜 이 결정이 필요했는가.
```

## 위생 규칙 (MD047 / MD012 / MD007)

렌더링을 깨뜨리진 않지만 diff 노이즈와 일관성 저하를 유발하는 것들이다. 모두 `--fix`로 자동 정리된다.

- **MD047 — 파일 끝 단일 개행:** 파일은 정확히 하나의 개행으로 끝난다. 끝 개행이 없거나 여러 개면 도구·diff가 마지막 줄을 어긋나게 표시한다.
- **MD012 — 연속 빈 줄 금지:** 빈 줄은 최대 1개. 의미 없는 빈 줄 누적을 막아 diff를 깔끔하게 유지한다.
- **MD007 — 중첩 리스트 들여쓰기 2칸:** 중첩 항목은 2칸 들여쓴다. 들여쓰기가 들쭉날쭉하면 일부 렌더러에서 중첩 깊이가 잘못 해석된다.
