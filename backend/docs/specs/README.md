# 기능 설계 명세 (specs)

`/design-first` 스킬이 비자명한 새 기능을 구현하기 전에 작성하는 **기능 단위 설계 명세**를 모은다. 승인된 명세는 `/impl`이 읽어 구현 기준으로 삼는다.

## ADR과의 경계

| 구분 | 다루는 것 | 수명 |
|------|-----------|------|
| `docs/adr/` | 모듈 간 의존, 패턴 도입 등 **범시스템·장기** 아키텍처 의사결정 | 장기 (결정이 유효한 동안) |
| `docs/specs/` | 한 기능의 명확화 결과·계층 설계·검증 가능한 성공 기준 등 **기능 단위·전술** 설계 | 단기 (기능 구현 동안) |

설계 중 **범시스템 아키텍처 결정**이 필요하면 명세가 아니라 `/adr`로 기록하고, 이 명세의 "관련 ADR"에 번호를 참조한다.

## 파일 규칙

- 파일명: `{kebab-case-기능명}.md`
- 형식: `/design-first` 스킬의 명세 템플릿을 따른다
- 상태: `초안` → (승인) → `승인`. 구현 완료 후의 갱신 정책은 팀이 정한다
