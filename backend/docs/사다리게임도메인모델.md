# 사다리 타기 게임 도메인

## LadderGame

### 상태

- `poles`: `Poles` 플레이어들이 배정된 기둥 목록
- `lines`: `LadderLines` 그려진 선 목록
- `bottomRanks`: `BottomRanks` 바닥의 순위 배치
- `state`: `LadderGameState` 현재 게임 상태
### 행위

- `startGame(List<Player>)`: 플레이어들을 기둥에 랜덤 배정하고 바닥 순위를 생성한다.
- `drawLine(PlayerName, int segmentIndex)`: 플레이어가 지정 구간에 선을 긋는다.
- `changeToDrawing()`: DRAWING 상태로 전환한다.
- `changeToResult()`: RESULT 상태로 전환한다.
- `changeToDone()`: DONE 상태로 전환한다.
- `getResult()`: 모든 플레이어의 경로를 추적하여 `MiniGameResult`를 반환한다.
- `getScores()`: 플레이어별 점수를 반환한다.
- `getMiniGameType()`: `LADDER_GAME`을 반환한다.
- `isAlreadyDrew(PlayerName)`: 해당 플레이어가 이미 선을 그었는지 확인한다. (`LadderLines.hasDrawn()` 위임)
- `tracePaths()`: 모든 플레이어의 시작 기둥 → 최종 기둥 인덱스를 계산한다.

### 규칙

- 선 긋기는 DRAWING 상태에서만 가능하다.
- 요청한 `PlayerName`이 `Poles`에 존재하는 참여자여야 한다. 미참여자 요청은 즉시 예외 처리한다.
- 플레이어당 최대 1개의 선만 그을 수 있다.
- `segmentIndex`는 0 이상 기둥 수 - 2 이하여야 한다.

## Poles

### 상태

- `poles`: `List<Pole>` 기둥 목록

### 행위

- `assign(List<Player>)`: 플레이어들을 기둥에 랜덤으로 배정한다.
- `getPoleIndex(PlayerName)`: 해당 플레이어의 기둥 인덱스를 반환한다.
- `getPlayer(int poleIndex)`: 해당 기둥 인덱스의 플레이어를 반환한다.
- `size()`: 기둥 수를 반환한다.
- `isValidSegment(int segmentIndex)`: 유효한 구간 인덱스인지 확인한다 (0 ≤ segmentIndex ≤ size - 2).

### 규칙

- 기둥 수는 플레이어 수와 동일하다.
- 플레이어-기둥 배정은 `startGame` 시 1회만 이루어진다.

- `player`: `Player` 배정된 플레이어

## LadderLines

### 상태

- `lines`: `List<LadderLine>` 그려진 선 목록 (삽입 순서 = 위→아래 순서)

### 행위

- `add(PlayerName, int segmentIndex)`: 새 선을 추가한다. 같은 구간 기존 선 아래, 인접 구간 동일 행 충돌 해소 후 배치한다.
- `hasDrawn(PlayerName)`: 해당 플레이어가 이미 선을 그었는지 확인한다.
- `trace(int startPoleIndex)`: 시작 기둥에서 출발해 선을 위→아래 순서로 따라가며 최종 기둥 인덱스를 반환한다.
- `size()`: 선의 개수를 반환한다.
- `getAll()`: 모든 선 목록을 반환한다.

### 규칙

- `add()`는 `synchronized`로 구현한다. row 계산과 삽입이 atomic해야 한다.
- 같은 행(row)에 인접한 두 구간의 선이 동시에 존재할 수 없다. 충돌 시 새 선의 row를 아래로 밀어 해소한다.
- 같은 구간에 새 선을 추가하면 해당 구간의 기존 선들보다 아래에 배치된다.
- 선의 삽입 순서가 수직 위치(row)를 결정한다. 먼저 삽입된 선이 위에 위치한다.

## LadderLine

_record_

### 상태

- `playerName`: `PlayerName` 선을 그은 플레이어 (브로드캐스트 및 RESULT 색깔 구분에 사용)
- `segmentIndex`: `int` 왼쪽 기둥 인덱스 (segmentIndex 기둥과 segmentIndex+1 기둥을 연결)
- `row`: `int` 수직 위치 (1이 최상단, 값이 클수록 아래)

## BottomRanks

### 상태

- `ranks`: `Map<Integer, Integer>` 기둥 인덱스 → 순위 (1-based)

### 행위

- `generate(int n)`: n개 기둥에 1~n 순위를 랜덤으로 배정하여 생성한다.
- `getRank(int poleIndex)`: 해당 기둥의 순위를 반환한다.
- `getAll()`: 전체 순위 배치 맵을 반환한다.

### 규칙

- 순위는 1~n까지 각 기둥에 중복 없이 정확히 하나씩 배정된다.

---

## LadderGameState

_Enum_

### 상수

- `DESCRIPTION`: 게임 규칙 설명
- `PREPARE`: 기둥 배정 및 바닥 순위 공개
- `DRAWING`: 선 긋기 (5초)
- `RESULT`: 경로 애니메이션 및 순위 공개
- `DONE`: 완료

### 상태

- `duration`: `int` 각 상태의 서버 지속 시간 (밀리초)

---

## LadderGameScore

_extends MiniGameScore_

### 상태

- `rank`: `int` 사다리 최종 순위 (1-based, 1등이 가장 낮은 커피 구매 확률 증가)

### 행위

- `getValue()`: 순위를 반환한다.
