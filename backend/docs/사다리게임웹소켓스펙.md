# 사다리 타기 게임 WebSocket 스펙

## 엔드포인트 목록

| 방향 | 경로 | 설명 |
|---|---|---|
| 구독 | `/topic/room/{joinCode}/ladder/state` | 게임 상태 전환 브로드캐스트 |
| 구독 | `/topic/room/{joinCode}/ladder/line` | 선 추가 브로드캐스트 |
| 발행 | `/app/room/{joinCode}/ladder/draw` | 선 그리기 요청 |

---

## 구독 메시지

### 상태 전환 — `/topic/room/{joinCode}/ladder/state`

#### DESCRIPTION

```json
{
  "state": "DESCRIPTION"
}
```

#### PREPARE

기둥 배정과 바닥 순위를 함께 전달한다. `poles` 배열의 순서가 화면 왼쪽부터 오른쪽 기둥 순서이다.

```json
{
  "state": "PREPARE",
  "poles": [
    { "index": 0, "playerName": "꾹이" },
    { "index": 1, "playerName": "철수" },
    { "index": 2, "playerName": "영희" }
  ],
  "bottomRanks": { "0": 2, "1": 1, "2": 3 }
}
```

#### DRAWING

`endTimeEpochMs`는 서버 기준 DRAWING 종료 시각(UTC epoch ms)이다. 클라이언트는 `Date.now()`와의
차이로 남은 시간을 계산한다.

```json
{
  "state": "DRAWING",
  "endTimeEpochMs": 1745640000000
}
```

#### RESULT

`animationDurationMs` 동안 경로 애니메이션을 재생한다. 서버도 동일 시간 후 DONE으로 전환한다.

```json
{
  "state": "RESULT",
  "rankings": {
    "꾹이": 1,
    "철수": 3,
    "영희": 2
  },
  "animationDurationMs": 5000
}
```

#### DONE

```json
{
  "state": "DONE"
}
```

---

### 선 추가 — `/topic/room/{joinCode}/ladder/line`

선 그리기 요청이 서버에서 확정된 후 전체 참여자에게 브로드캐스트된다.

`segmentIndex`는 왼쪽 기둥 인덱스이며, `segmentIndex` 기둥과 `segmentIndex + 1` 기둥을 연결한다.
`row`는 수직 위치로 1이 최상단이며 값이 클수록 아래에 위치한다.

```json
{
  "playerName": "꾹이",
  "segmentIndex": 2,
  "row": 3
}
```

---

## 발행 메시지

### 선 그리기 — `/app/room/{joinCode}/ladder/draw`

`segmentIndex`는 0 이상 (기둥 수 - 2) 이하여야 한다. 서버는 검증 실패 시 응답 없이 무시한다.

```json
{
  "playerName": "꾹이",
  "segmentIndex": 2
}
```

---

## 낙관적 업데이트 처리

선 그리기는 row를 서버에서 결정하므로 완전한 낙관적 업데이트가 불가능하다.
대신 아래 흐름으로 즉각적인 피드백을 제공한다.

```text
1. 클라이언트가 segmentIndex 터치
2. 해당 segment에 ghost 선(반투명) 즉시 표시
3. 서버로 draw 메시지 발행
4. /ladder/line 브로드캐스트 수신
   - 내 playerName 포함 → ghost를 확정 선으로 교체, 실제 row에 배치
   - 2초 내 미수신 (검증 실패) → ghost 제거
```

row가 ghost 위치와 다를 경우 선을 실제 row로 이동한다.
지연이 짧을수록(통상 200ms 이하) 이동이 눈에 띄지 않는다.

---

## 상태 전환 타이밍

```text
DESCRIPTION  ──5초──▶  PREPARE  ──3초──▶  DRAWING  ──5초──▶  RESULT  ──5초(고정)──▶  DONE
```

서버와 클라이언트 모두 RESULT 수신 시 `animationDurationMs` 값으로 타이머를 동기화한다.

---

## 유효성 검증 실패 처리

서버는 아래 케이스에서 응답 없이 요청을 무시하고 warn 로그만 기록한다.
클라이언트는 브로드캐스트 미수신으로 실패를 암묵적으로 감지한다.

| 케이스 | 서버 동작 | 클라이언트 동작 |
|---|---|---|
| 이미 선을 그은 플레이어 재요청 | 무시 + warn | ghost 제거 (2초 타임아웃) |
| 유효하지 않은 segmentIndex | 무시 + warn | ghost 제거 (2초 타임아웃) |
| DRAWING 상태가 아닐 때 요청 | 무시 + warn | ghost 제거 (2초 타임아웃) |

---

## RESULT 애니메이션 가이드

RESULT 수신 시 `rankings`와 클라이언트가 이미 보유한 선 데이터로 경로를 계산하고 애니메이션을 재생한다.

```text
1. 각 플레이어의 시작 기둥(PREPARE에서 수신)에서 출발
2. row 1부터 maxRow까지 순서대로
   - 현재 기둥이 선의 segmentIndex 또는 segmentIndex+1에 해당하면 반대쪽 기둥으로 이동
3. 최종 기둥의 bottomRank = 해당 플레이어 순위
4. 모든 플레이어 경로를 동시에 animationDurationMs에 걸쳐 재생
5. 애니메이션 종료 후 rankings 결과 화면 표시
```

`rankings`는 서버 확정값이므로 클라이언트 경로 계산 결과와 반드시 일치해야 한다.
불일치 시 `rankings` 값을 우선한다.
