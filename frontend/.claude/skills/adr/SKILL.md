# ADR (Architecture Decision Record) 스킬

## 목적

구현에 앞서 **왜 이 방향을 선택했는지**를 문서화한다.
코드는 무엇을 했는지 보여주지만, 결정 근거는 코드에서 추론할 수 없다.

## 프로세스

이 스킬이 로드되면 다음 순서를 따른다.

### 1단계 — 논의 (아직 충분히 논의되지 않은 경우)

아래 항목이 불명확하면 질문으로 명확히 한다:

- **문제**: 무엇을 해결하려 하는가?
- **제약**: 기존 코드·API·일정 제약이 있는가?
- **대안**: 다른 접근법을 검토했는가?

이미 대화에서 논의된 내용은 다시 묻지 않는다.

### 2단계 — 기존 ADR 충돌 확인

`docs/adr/` 의 기존 ADR 파일 목록을 읽고, 현재 논의 중인 결정과 관련 있는 파일을 식별한다.

관련 ADR이 있으면 읽고 다음을 확인한다:

- **직접 충돌**: 기존 ADR에서 채택한 방향을 현재 결정이 뒤집는가?
- **전제 충돌**: 기존 ADR의 "결과 및 영향"에서 언급한 주의사항을 현재 결정이 위반하는가?
- **기각된 대안 재채택**: 기존 ADR에서 명시적으로 기각한 접근법을 현재 결정이 다시 선택하는가?

충돌이 발견되면 구현으로 넘어가기 전에 사용자에게 명시한다:

```
⚠️ 기존 ADR과 충돌
파일: docs/adr/{파일명}
충돌 내용: {기존 결정 요약}
현재 방향: {충돌하는 이유}
→ 기존 ADR을 supersede할지, 현재 방향을 수정할지 확인이 필요합니다.
```

기존 ADR을 supersede하기로 결정한 경우 새 ADR에 다음을 추가한다:

```md
## 대체하는 ADR

[{기존 ADR 제목}](../adr/{파일명}) 을 대체한다.
대체 이유: {왜 이전 결정을 바꾸는가}
```

그리고 기존 ADR의 frontmatter `status`를 `superseded`로 업데이트한다.

관련 ADR이 없거나 충돌이 없으면 다음 단계로 바로 진행한다.

### 3단계 — 결정 확인

구현 전에 결정 방향을 한 문장으로 명시하고 사용자에게 확인한다.

### 4단계 — ADR 파일 저장

`docs/adr/YYYYMMDD-{kebab-slug}.md` 형식으로 저장한다.
날짜는 오늘 날짜(현재 시스템 날짜)를 사용한다.

```md
---
date: YYYY-MM-DD
status: decided
---

# {제목}

## 맥락

{어떤 문제·요구사항·제약에서 비롯된 결정인가. 배경을 모르는 사람도 이해할 수 있게.}

## 결정

{무엇을 선택했는가 — 핵심 방향을 한두 문장으로.}

## 고려한 대안

- **{대안 A}**: {기각 이유}
- **{대안 B}**: {기각 이유}

## 결과 및 영향

{코드 변경 범위, 트레이드오프, 향후 주의사항}
```

ADR 저장 후 파일 경로를 명시하고 구현으로 이어간다.

### 5단계 — 구현

ADR에 기록된 결정에 따라 구현한다.
구현 중 방향이 바뀌면 ADR을 업데이트한다.

---

## ADR을 쓰지 않는 경우

결정 근거가 자명한 작업은 ADR 없이 바로 구현한다:

- 스타일·타입 수정
- 버그 수정 (원인이 명확한 경우)
- 오타·포맷 수정
- 문서 업데이트

**기준**: "왜 이렇게 했지?"라는 질문이 나올 만한 작업이면 ADR을 쓴다.

## 파일 명명 규칙

| 항목   | 형식                                      | 예                               |
| ------ | ----------------------------------------- | -------------------------------- |
| 경로   | `docs/adr/`                               |                                  |
| 파일명 | `YYYYMMDD-kebab-slug.md`                  | `20260515-websocket-rst-auth.md` |
| slug   | 결정 핵심 키워드 3–5단어, 영어 kebab-case | `ranking-tab-animation-strategy` |

## qmd 연동

`docs/adr/`는 `zzol-docs` 컬렉션에 인덱싱된다.
이후 세션에서 과거 결정을 조회할 때:

```bash
qmd query "WebSocket 인증 방식 결정" --collection zzol-docs
qmd query "랭킹 탭 구조 선택 근거" --collection zzol-docs
```
